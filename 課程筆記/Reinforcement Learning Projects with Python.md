# Chapter 1. Reinforcement Learning Fundamentals

---

## What is Reinforcement Learning?

Reinforcement Learning (RL)

是一種透過與環境互動來學習最佳決策策略的方法。

Agent 不斷：

1. 觀察環境
2. 採取行動
3. 獲得回饋
4. 更新策略

最終最大化長期累積獎勵。

---

## RL Components

### Agent

決策者

例如：

- 機器人
- 自駕車
- AI 玩家

---

### Environment

環境

例如：

- 遊戲世界
- 工廠系統
- 網路系統

---

### State (S)

系統當前狀態

例如：

- 玩家位置
- 機械設備狀態

---

### Action (A)

Agent 可執行動作

例如：

- 上下左右移動
- 調整加工參數

---

### Reward (R)

環境回饋

例如：

+100 成功

-10 失敗

---

### Policy (π)

策略

定義：

```text
State → Action
```

---

### Value Function

評估某個狀態的價值

```text
V(s)
```

---

### Q Function

State + Action 的價值

```text
Q(s,a)
```

# Chapter 4. Bellman Equations

---

# 強化學習的靈魂：從 Bellman 方程到現代頂級 AI

## 核心本質
> 「一個狀態有多好，不看現在，而看從這裡開始未來能獲得多少獎勵。」
> 整個 Reinforcement Learning (RL)，其實就是不斷利用 Bellman Equation 去估計未來、更新價值、找到最佳決策的過程。

---

## 一、 什麼是 Bellman Equation？
**【白話解讀】老謀深算的記帳方式**

真正聰明的人，看的是長遠的「預期總回報」，而不是眼前的蠅頭小利。



* **眼前的獎勵：** 今天中午吃大餐，當下很爽（即時獎勵 $+100$ 分）。
* **未來的價值：** 下午忍痛去補習班上課，當下很累（即時獎勵 $-50$ 分），但這個決定會讓你明天考試及格、拿到零用錢、未來考上好大學（未來的潛在價值 $+1000$ 分）。

$$某個抉擇的真正總價值 = 現在馬上下肚的獎勵 + 待會從新起點出發後，未來預計還能賺到的所有獎勵（打個折 \gamma）$$

---

## 二、 AI 的進化史：小明的一天

為了處理越來越複雜的世界，AI 的演算法（從 Q-Learning 到 SAC）不斷升級，但它們底層的 **Bellman 思想**從未改變。

### 1. Q-Learning（小學階段：死記硬背）
* **小明的工具：** 一本**實體筆記本（Q-Table）**。
* **白話運作：** 小明剛搬到新城市，手裡拿著一本空白筆記本。
    * **第一天：** 肚子餓，在十字路口（狀態）隨便選了往左走（動作），吃到美味臭豆腐，獲得 $100$ 分。他翻開筆記本寫下：「路口往左 = $100$ 分」。
    * **第二天：** 隔壁飲料店老闆告訴他，如果吃完臭豆腐再往前走兩步，有超級冰淇淋（預估未來價值 $200$ 分）。小明立刻翻開筆記本，用 Bellman 思想把紀錄更新為：「路口往左 = $100$（當下） + $200$（未來） = $300$ 分」。
* **缺點：** 太死腦筋。如果城市有 300 萬個路口，筆記本會重到搬不動（記憶體爆掉）；遇到沒去過的新路口，直接當機。

### 2. DQN（中學階段：融會貫通）
* **小明的工具：** 一台**超級計算機（深度神經網路）**。
* **白話運作：** 小明把沉重的筆記本丟了。現在遇到全新、沒去過的路口，他只要把路口的特徵（有紅綠燈、有捷運站）輸入計算機，計算機就會自動**「找規律並盲猜」**出一個未來價值（例如：猜 50 分）。
    * **怎麼變聰明？** 小明實際走過去後，發現依據 Bellman 公式算出來的標準答案應該是 $300$ 分。小明就會對計算機大喊：「你猜太低了！給我修正內部電路（更新權重 Weight）！」
* **優勢：** 再多、再複雜的格子都不怕，AI 學會了「舉一反三」。

### 3. PPO (**P**roximal **P**olicy **O**ptimization)/ SAC(**S**oft **A**ctor-**C**ritic)（大學階段：高手級分工）
當小明長大要去學開賽車時，動作變得極度精細（油門踩 35% 還是 36%？方向盤轉 12.5 度還是 13 度？）。這種連續動作，DQN 計算機按到冒煙也算不出來。於是，大腦內部啟動了**「組織分工（Actor-Critic 架構）」**：
* **記帳員（Critic）：** 負責用 Bellman 思想一直算現在和未來的分數。
* **賽車手（Actor）：** 負責靈活、精細地轉動方向盤與踩油門。記帳員會一邊看、一邊碎念賽車手：「你剛剛那一踩油門，未來的預期分數變低了喔！」賽車手聽到就馬上修正。

#### 兩大現代高手演算法的獨門絕招：
* **PPO（近端策略優化）：** 穩健派。限制賽車手每次改變開車策略的幅度不能超過 20%。循序漸進，安全第一，不會因為一次失誤就徹底學壞（OpenAI 訓練 GPT 常用此技術）。
* **SAC（柔性演員-評論家）：** 好奇派。在賺錢的同時，鼓勵賽車手大膽探索世界，越好玩、越隨機越好。這種絲滑的控制力，是目前機械手臂和自駕界頂級大師。

---

## 三、 演算法進化總覽表

| 演算法階段 | 小明的工具 | 怎麼做決策 | 適合玩什麼 |
| :--- | :--- | :--- | :--- |
| **Q-Learning** | 一本**實體筆記本** | 翻到那一頁，看哪一個分數高就照著做。 | 走迷宮、井字棋（格子極少的環境） |
| **DQN** | 一台**超級計算機** (神經網路) | 遇到新狀況，用按計算機的方式**猜**出未來價值。 | Atari 電玩、打磚塊 (動作非 A 即 B) |
| **PPO / SAC** | **兩個人分工** (賽車手+記帳員) | 記帳員用 Bellman 算價值，賽車手負責把連續動作做到極致精細。 | 開賽車、機器人控制、操作無人機 |

## 總結
不論演算法的名字變得多酷炫，它們底層在做的事情從來沒有變過：
**「根據昨天的經驗，加上今天的遭遇，去重新估算明天的價值，進而做出最好的決定。」** 這就是 Bellman 思想，也是整個強化學習的基石。

---

### Inspiration

---

### Acceptance Probability

這個公式是 **Simulated Annealing (SA，模擬退火法)** 的核心接受機率公式（Acceptance Probability）：

P=e−ΔE/TP = e^{-\Delta E/T}P=e−ΔE/T

其中：

|符號|全名|意義|
|---|---|---|
|PPP|Acceptance Probability|接受較差解的機率|
|eee|Euler's Number|自然常數 ≈ 2.71828|
|ΔE\Delta EΔE|Energy Difference|新解比舊解差多少|
|TTT|Temperature|系統溫度|

---

## 為什麼需要這個公式？

一般最佳化演算法：

```
如果新解更好→ 接受如果新解更差→ 拒絕
```

缺點：

容易卡在 Local Optimum（局部最佳解）。

例如：

```
      Global Optimum          ↓         /\        /  \       /    \      /      \     /        \    /\        /   /  \______/ ← Local Optimum
```

如果只接受更好的解：

可能永遠爬不上最高峰。

---

## Simulated Annealing 的想法

模仿金屬退火：

高溫時：

```
原子亂跳
```

低溫時：

```
逐漸穩定
```

因此：

即使找到較差解，

也有機率接受。

這樣才能跳出局部最佳解。

---

## ΔE 的意思

假設：

目前解：

```
Cost = 100
```

新解：

```
Cost = 105
```

那麼：

ΔE=105−100\Delta E = 105 - 100ΔE=105−100 ΔE=5\Delta E = 5ΔE=5

表示：

新解比較差。

---

## 高溫時

假設：

T=100T = 100T=100 ΔE=5\Delta E = 5ΔE=5

則：

P=e−5/100P=e^{-5/100}P=e−5/100 P=e−0.05P=e^{-0.05}P=e−0.05 P≈0.951P \approx 0.951P≈0.951

接受機率：

95.1%

幾乎會接受。

---

## 低溫時

假設：

T=1T=1T=1 ΔE=5\Delta E=5ΔE=5

則：

P=e−5P=e^{-5}P=e−5 P≈0.0067P \approx 0.0067P≈0.0067

接受機率：

0.67%

幾乎不接受。

---

## 圖形直觀

使用這個函數：

y=e−xy=e^{-x}y=e−x

可以看到：

```
x 越大↓e^-x 越小
```

因此：

```
ΔE 越大（解越差）↓接受機率越小
```

而：

```
T 越大↓ΔE/T 越小↓接受機率越高
```
