# Chapter 1 - Why C/C++ Programs Become Vulnerable

---

## 為什麼 C/C++ 特別容易出現漏洞

C/C++ 具有：

- Low-Level Hardware Access
- Manual Memory Management
- Pointer Manipulation
- Direct System Interaction

這些能力帶來高效能，同時也帶來高風險。

---

## Manual Memory Management

### 定義

程式員自行管理：

```
malloc()free()newdelete
```

---

### 風險

- Memory Leak
- Use After Free
- Double Free
- Dangling Pointer

---

## Pointer Pitfalls

Pointer 是 C/C++ 最大優勢也是最大風險。

例如：

```
int *ptr = nullptr;*ptr = 10;
```

產生：

```
Segmentation Fault
```

---

### 常見問題

#### Null Pointer Dereference

```
ptr == nullptr
```

仍然存取。

---

#### Dangling Pointer

```
free(ptr);ptr->value;
```

---

#### Wild Pointer

未初始化 Pointer。

---

## Low-Level System Access

C/C++ 可以：

- 直接存取記憶體
- 操作 Device Driver
- 控制 Embedded System
- Kernel Programming

若控制錯誤：

可能導致：

- Arbitrary Code Execution
- Privilege Escalation
- System Crash

---

## Risky Type Conversions

### Implicit Conversion

```
int x = -1;unsigned int y = x;
```

結果：

```
4294967295
```

---

### 問題

造成：

- Integer Overflow
- Integer Underflow
- Buffer Miscalculation

---

## Dangerous Library Functions

---

### gets()

危險程度：

★★★★★

```
gets(buffer);
```

無法限制輸入長度。

---

### strcpy()

```
strcpy(dest, src);
```

可能造成：

```
Buffer Overflow
```

---

### strcat()

```
strcat(dest, src);
```

容易超出容量。

---

### sprintf()

```
sprintf(buffer,"%s",input);
```

可能覆蓋記憶體。

---

## Legacy Code Risks

老舊程式碼常出現：

- Unsafe APIs
- No Input Validation
- Hardcoded Credentials
- Poor Error Handling

---

# Chapter 2 - Common Security Vulnerabilities

---

# Buffer Overflow

最著名漏洞之一。

---

## Stack Overflow

```
char buf[10];strcpy(buf,input);
```

輸入：

```
AAAAAAAAAAAAAAAAAAAA
```

超出範圍。

---

可能覆寫：

- Return Address
- Saved Frame Pointer

---

### 後果

```
Remote Code Execution (RCE)
```

---

# Heap Overflow

發生於：

```
malloc()new
```

配置區域。

---

攻擊者可修改：

- Heap Metadata
- Function Pointer
- Virtual Table (vtable)

---

# Integer Overflow

---

## 範例

```
uint8_t x = 255;x++;
```

結果：

```
0
```

---

### 危害

可能造成：

```
malloc(size + 1)
```

實際配置空間不足。

---

# Format String Vulnerability

---

## 危險寫法

```
printf(userInput);
```

攻擊者輸入：

```
%x %x %x %x
```

即可讀取記憶體。

---

### 更危險

```
%n
```

可寫入記憶體。

---

### 正確寫法

```
printf("%s", userInput);
```

---

# Improper Error Handling

---

## 問題

忽略錯誤：

```
FILE* fp = fopen("test.txt","r");fgets(buffer,100,fp);
```

若：

```
fp == NULL
```

則程式崩潰。

---

### 正確做法

```
if(fp == NULL){    return ERROR;}
```

---

# Race Condition

---

## TOCTOU

Time Of Check To Time Of Use

---

範例：

```
if(access(file,W_OK)==0){    fopen(file,"w");}
```

檢查與使用之間被攻擊者替換。

---

後果：

```
Privilege Escalation
```

---

# Chapter 3 - Secure Coding Principles

---

# Attack Surface Reduction

---

## Attack Surface

系統暴露給攻擊者的入口。

例如：

- Open Ports
- APIs
- File Upload
- Database Interface

---

### 原則

只保留必要功能。

---

# Principle of Least Privilege (PoLP)

---

使用最小權限。

---

錯誤：

```
Run as Administrator
```

---

正確：

```
Run as Standard User
```

---

# Fail-Safe Defaults

---

預設拒絕。

---

錯誤：

```
Allow All
```

---

正確：

```
Deny by Default
```

---

# Defense in Depth

多層防護。

---

例如：

```
Firewall↓WAF↓Authentication↓Authorization↓Encryption↓Logging
```

# Chapter 4 - Secure Memory Management

---

# C Memory Management

---

## malloc()

```
int* p = malloc(sizeof(int));
```

---

## free()

```
free(p);
```

---

# Memory Leak

---

配置後未釋放。

```
while(true){    malloc(100);}
```

---

造成：

```
Denial of Service (DoS)
```

---

# Double Free

---

```
free(ptr);free(ptr);
```

---

可能造成：

```
Heap Corruption
```

---

# Use After Free (UAF)

---

```
free(ptr);ptr->value = 10;
```

---

屬於高危漏洞。

---

可能導致：

```
Remote Code Execution
```

---

# Null Pointer Dereference

---

```
ptr = nullptr;ptr->data;
```

---

造成：

```
Application Crash
```

---

# C++ Memory Management

---

# new

```
MyClass* obj = new MyClass();
```

---

# delete

```
delete obj;
```

---

# new[]

```
int* arr = new int[10];
```

---

# delete[]

```
delete[] arr;
```

---

## Memory Mismatch

錯誤：

```
newfree
```

或

```
mallocdelete
```

---

會造成：

```
Undefined Behavior
```

---

# Smart Pointer

---

C++11 之後推薦。

---

## unique_ptr

```
std::unique_ptr<int>
```

單一擁有者。

---

## shared_ptr

```
std::shared_ptr<int>
```

引用計數。

---

## weak_ptr

避免：

```
Circular Reference
```

---

### 優點

- Automatic Cleanup
- RAII
- Safer Code

# Chapter 6 - Secure Coding Checklist

---

## Input Validation

- Validate Length
- Validate Type
- Validate Range
- Validate Format

---

## Memory Safety

- Avoid Raw Pointer
- Prefer Smart Pointer
- Free Allocated Memory
- Avoid Double Free

---

## String Handling

避免：

```
gets()strcpy()strcat()sprintf()
```

改用：

```
fgets()strncpy()strncat()snprintf()
```

---

## Error Handling

- Check Return Values
- Handle Exceptions
- Log Errors

---

## Access Control

- Least Privilege
- Authentication
- Authorization

---

## Secure Design

- Defense in Depth
- Fail-Safe Defaults
- Secure by Design
